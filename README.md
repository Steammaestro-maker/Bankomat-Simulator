# 🏦 ATM Simulator — Симулятор Банкомата

Полнофункциональный симулятор банкомата на Python с поддержкой регистрации карт, 
аутентификации и операций со счётом.

## ✨ Особенности

- ✅ Регистрация новых банковских карт
- ✅ Автоматическая генерация номера карты, срока действия и CVV
- ✅ Защищённый ввод PIN (скрытый в TTY)
- ✅ Аутентификация с ограничением попыток (макс. 3)
- ✅ Операции: просмотр баланса, пополнение, снятие
- ✅ История транзакций
- ✅ Смена PIN-кода
- ✅ Сохранение данных в JSON файл
- ✅ Русскоязычный интерфейс
- ✅ Подтверждение каждой операции (д/н)

## 🚀 Установка

### Требования
- Python 3.7+
- Стандартная библиотека Python (все модули встроены)

### Запуск программы

1. Клонируйте репозиторий или скачайте файл `atm_simulator.py`
2. Откройте терминал в директории проекта
3. Выполните команду:

```bash
python3 atm_simulator.py
```

или на Windows:

```bash
python atm_simulator.py
```

## 📖 Использование

### Главное меню

```
1. Зарегистрировать новую карту
2. Просмотреть зарегистрированные карты
help. Справка
0. Выйти
```

### Регистрация карты

1. Выберите пункт **1**
2. Введите имя и фамилию
3. Система автоматически сгенерирует:
   - 16-значный номер карты
   - Срок действия (MM/YY)
   - CVV-код (3 цифры)
4. Установите PIN (4-6 цифр)
5. Повторите PIN для подтверждения
6. Подтвердите регистрацию

**Начальный баланс**: 10,000 сом

### Вход в карту

1. Выберите пункт **2**
2. Выберите карту из списка
3. Введите PIN (скрытый ввод)
4. При неверном PIN остаётся 3 попытки

### Меню карты

После входа доступны операции:

```
1. Просмотреть баланс
2. Пополнить счёт
3. Снять наличные
4. История транзакций
5. Сменить PIN
6. Показать данные карты
help. Справка
0. Выйти из сессии
```

#### Детали операций

**1️⃣ Баланс**
- Просмотр текущего остатка на счёте
- Валюта: сом

**2️⃣ Пополнение**
- Введите сумму (целое число)
- Баланс увеличится на эту сумму
- Операция записывается в историю

**3️⃣ Снятие**
- Введите сумму (не более текущего баланса)
- Если средств недостаточно, операция отменяется
- История пополняется записью о снятии

**4️⃣ История**
- Просмотр всех транзакций (пополнения и снятия)
- Формат: `тип операции: сумма сом`

**5️⃣ Смена PIN**
- Введите текущий PIN (скрытый ввод)
- Введите новый PIN (видимый ввод, 4-6 цифр)
- PIN должен состоять только из цифр

**6️⃣ Данные карты**
- Полное имя владельца
- Номер карты (последние 4 цифры видны)
- Срок действия
- CVV-код

## 💾 Сохранение данных

Все данные карт сохраняются в файл **`cards.json`** в той же директории, что и программа.

Структура файла:
```json
{
  "cards": [
    {
      "number": "4536123456789012",
      "first_name": "Иван",
      "last_name": "Иванов",
      "pin": "1234",
      "expiry": "12/28",
      "cvv": "456",
      "balance": 10000,
      "transactions": []
    }
  ]
}
```

## 🔒 Безопасность

- 🔐 PIN вводится скрытым способом (функция `getpass()`)
- 🚫 Максимум 3 попытки ввода PIN
- ✅ Валидация всех числовых данных
- 📝 Все операции требуют подтверждения пользователя
- 📋 Полная история транзакций

## 🐛 Возможные ошибки и решения

| Ошибка | Решение |
|--------|---------|
| `cards.json не найден` | Файл создаётся автоматически при первой регистрации |
| `Превышено количество попыток` | Верните 5-10 минут и попробуйте снова (в коде можно увеличить `MAX_PIN_ATTEMPTS`) |
| `Недостаточно средств` | Пополните счёт через пункт "Пополнить счёт" |
| `PIN должен состоять из 4-6 цифр` | Используйте только цифры, длина 4-6 символов |

## 🏗️ Архитектура кода

### **Класс Card (dataclass)**
Хранит данные карты: имя владельца, фамилию, номер, PIN, срок действия и CVV.

### **Класс ATM**
Основной класс для работы с банкоматом. Управляет:
- Аутентификацией (проверка PIN)
- Операциями: баланс, пополнение, снятие
- Историей транзакций
- Сменой PIN

### **Функции сохранения данных**
- `load_data()` — загрузка карт из JSON
- `save_data()` — сохранение карт в JSON
- `create_card_record()` — создание записи карты
- `update_card_record()` — обновление данных

### **Генератор данных карт**
- `generate_card_number()` — уникальный номер (16 цифр)
- `generate_expiry()` — срок действия (2-7 лет от текущей даты)
- `generate_cvv()` — трёхзначный код безопасности

### **Интерактивные функции**
- `register_card_interactive()` — регистрация новой карты с валидацией
- `select_card_and_auth()` — выбор карты и аутентификация (3 попытки)
- `session_loop()` — основное меню работы с картой
- `prompt_yes_no()` — запрос подтверждения (д/н)
- `read_pin()` — защищённый ввод PIN через `getpass()`
